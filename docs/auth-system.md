# Authentication System

## Login Flow
1. Client submits credentials via login form.
2. Server action `loginAction()` validates credentials against DB.
3. Server returns artificially generated JWT via Base64 encoded JSON map.
4. Client `AuthProvider` persists token map into `sessionStorage`.
5. Route redirection logic triggers based on role boundary.

## Token Structure
```typescript
interface AuthTokenPayload {
  sub: string;       // User ID
  email: string;     // User Email
  role: string;      // UserRole ('admin', 'teacher', 'student')
  name: string;      // User Display Name
  createdAt: string; // ISO Timestamp
  exp: number;       // Expiry (Unix timestamp Date.now() + 24H)
}
```

## Storage
- **Mechanism:** `sessionStorage` (volatility: per browser tab)
- **Key:** `learners_academy_auth`

## Hydration Flow
1. App Mount. `AuthProvider` triggers.
2. Parses `learners_academy_auth` from `sessionStorage`.
3. Validates `exp` boundary. Fails silent if invalidated + purges state.
4. Loads into global React State `user` object.

## Route Protection Logic
- **Execution:** Happens internally inside `AuthProvider` `useEffect`.
- **Validation:** Matches `pathname` against `/admin`, `/teacher`, `/student`.
- **Enforcement:** If `!isAuthenticated`, forces redirect to `/auth/login`. If Role Mismatch, forces redirect to absolute root of respective assigned portal.

## Identifier Definitions
- `user.id`: The 32-bit CUID generated by Prisma `id` field. **MUST** be used as the definitive relational foreign key. Populates token `sub`.
- `user.studentId`: A human-readable legacy institutional identifier (e.g., 'STU-1002'). **MUST NOT** be used for data relationship mapping. Only for display.
- `user.email`: Institutional email. Defines unique login identity constraint.

---
NEVER USE `undefined` USER DATA FOR CRITICAL LOGIC. State resolution explicitly requires verification `if (!user || user.id === undefined) return;`
